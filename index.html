Mẹo dùng Codex hiệu quả

01

Scope nhỏ

1 prompt = 1 task = 1 commit. Đừng "build cả game" — yêu cầu "thêm component spawner". Dễ review, dễ rollback.

02

AGENTS.md

File ngữ cảnh project ở root repo. Codex auto-load: convention, kiến trúc, lệnh test. Viết 1 lần, dùng mãi.

03

Đính file bằng @

Prompt: fix bug trong @Spawner.cs. Codex không phải grep cả repo — nhanh và chính xác hơn.

04

Review diff luôn

Codex show patch trước khi apply. Đừng accept mù — đọc kỹ, nhất là khi nó "tự ý" sửa file ngoài scope.

05

Gate lệnh nguy hiểm

Set approval = gated cho rm -rf, git push --force, migration DB. Auto-approve chỉ với read/build/test.

06

Clear khi loạn

Context dài → AI hallucinate, lặp sai. /clear + restart session. Đừng "cố vớt vát" — tốn thời gian hơn.

6 mẹo này tôi đúc kết sau ~6 tháng dùng Codex daily. Nhấn mạnh tip 5 (gated) — đã thấy người để full-auto rồi Codex rm nhầm folder. Tip 2: AGENTS.md là "system prompt" của riêng project — đầu tư 30 phút viết, tiết kiệm hàng giờ sau.

Best Practices

  1. Scope nhỏ mỗi prompt — 1 task = 1 commit, dễ review diff.
  2. Skill > Prompt dài — nếu repeat 2+ lần, viết thành skill.
  3. Cho Codex chạy test — đừng chỉ generate code, để nó verify.
  4. Review diff như review PR — Codex sai vẫn sai, bạn là gatekeeper.
  5. Commit thường xuyên — checkpoint để rollback an toàn.
  6. Skill là tài sản team — sync vào repo, version control, PR review.
Điểm 6 quan trọng nhất: skill là code, không phải config cá nhân. Mỗi gotcha team gặp → cập nhật skill → tất cả member hưởng lợi.

Pitfalls — Tránh điều này

❌ Tin tuyệt đối

Codex hallucinate API name, nhất là package mới. Luôn chạy build/test.

❌ Prompt mơ hồ

"Làm game đẹp hơn" → vô dụng. "Thêm screen shake khi hit enemy" → actionable.

❌ Auto-merge

Không bao giờ để Codex tự push thẳng main. PR + review luôn.

❌ Bỏ qua skill update

Sửa cùng 1 lỗi 2 lần = skill có gap. Update skill ngay, đừng "để sau".

Mỗi pitfall đều có thật trong quá trình tôi maintain oh-my-game-kit.

3 điều mang về

1

Codex = tay AI

Không chỉ chat — đọc, sửa, chạy, test.

2

Skill = context

Đầu tư vào skill kit thay vì prompt dài.

3

Bạn vẫn là dev

Review, test, ship. AI tăng tốc, không thay thế.
